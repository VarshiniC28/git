Topic: Syncing commits between old and new cloned repositories (local-to-local)

Suppose you clone an existing remote repository (Repo A) into a new folder on your machine (creating Repo B). You make some changes in Repo B, commit them, and push them to the remote repository.

Now, when you run git log in the new local repository (Repo B), you will see both the old commits and your new commits. The changes are also visible on the remote repository because you pushed them.

However, if you go back to the old local repository (Repo A) and check the logs using git log, you wonâ€™t see the new commits made from Repo B immediately.

To update the old local repository (Repo A) with the latest commits from the remote, run: 

git fetch

After fetching, you can see the new commits using: 

git log origin/main

This shows the updated commit history from the remote (which was pushed by the new repo).

Summary:

  • Commits are stored locally unless pushed.
  • Each cloned repo is independent but connected to the same remote.
  • To sync another local repo, use git fetch to get remote changes.
